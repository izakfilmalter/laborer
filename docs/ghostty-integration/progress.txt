- Created ghostty-host.ts: standalone helper process following the PTY Host pattern
  - Communicates via newline-delimited JSON over stdin (commands) and stdout (events)
  - Loads the native @laborer/ghostty addon on startup
  - Initializes Ghostty runtime, creates app, starts periodic tick timer (~60fps)
  - Accepts commands: create_surface, destroy_surface, set_size, set_focus, get_iosurface, get_size, list_surfaces
  - Emits events: ready (with version), surface_created, surface_destroyed, size_result, iosurface_result, surfaces_list, ok, error
  - All commands use request IDs for response correlation
  - Graceful shutdown on stdin close, SIGTERM, SIGINT
  - Debug logging to stderr (not part of IPC protocol)
- Created ghostty-host-client.ts: Effect Service for communicating with the helper process
  - GhosttyHostClient Context.Tag with Layer.scoped lifecycle
  - Spawns ghostty-host.ts as child process (tsx in dev, node in bundled mode)
  - Waits for ready event via Effect.Deferred before accepting commands
  - Request/response pattern with Effect.async and pending request tracking
  - Monotonic command ID generation for response correlation
  - Crash detection with callback registration
  - Rejects all pending requests on process exit
  - Automatic cleanup via Effect.addFinalizer on layer teardown
  - Full typed API: createSurface, destroySurface, setSurfaceSize, setSurfaceFocus, getIOSurfaceId, getSurfaceSize, listSurfaces, onCrash
- Added package.json export for "./client" entry point
- Added effect and tsx as dependencies

  - Startup: emits ready event with Ghostty version

  - Multi-surface: list surfaces after creating two
  - Error handling: invalid JSON, invalid command structure
  - Graceful shutdown: clean exit when stdin closes

### Key decisions
- Mouse button mapping accounts for the browser→Ghostty mismatch: browser uses 0=main,1=aux,2=secondary while Ghostty uses 1=left,2=right,3=middle
- Mouse events use fire-and-forget IPC (same as keyboard) — lost events are acceptable since terminal input is best-effort
- Context menu is prevented within the terminal surface — right-click is forwarded to Ghostty as a mouse button event for terminal apps that need it
- Scroll events forward raw deltaX/deltaY from browser WheelEvent — Ghostty handles the actual scrollback interpretation
- scrollMods is 0 for standard wheel events — precision trackpad/momentum scrolling is not yet wired from the browser
- Mouse position is computed relative to the container element's bounding rect (clientX - rect.left) for correct surface-local coordinates
- Mouse position is sent alongside button press events since Ghostty needs both the position and button state for correct click handling
