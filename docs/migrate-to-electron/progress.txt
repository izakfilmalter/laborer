### Issue 7: Shell environment probing + port allocation — DONE
- Created `apps/desktop/src/shell.ts` — shell environment probing utilities:
  - `extractPathFromShellOutput()` — extracts PATH value from sentinel-delimited shell output
  - `readPathFromLoginShell()` — launches user's login shell with `-ilc` to capture full PATH
  - Uses `__LABORER_PATH_START__` / `__LABORER_PATH_END__` sentinel markers to safely ignore shell startup noise (fish greeting, motd, etc.)
  - Injectable `execFile` parameter for testability
  - 5-second timeout to prevent hangs from broken shell configs
- Created `apps/desktop/src/fix-path.ts` — macOS PATH fix:
  - `fixPath()` called synchronously at module top-level in main.ts, before any child processes
  - On macOS, reads user's default shell from `$SHELL` (falls back to `/bin/zsh`)
  - Silently catches errors to prevent broken shell configs from blocking app launch
  - No-op on non-macOS platforms
- Created `apps/desktop/src/ports.ts` — ephemeral port reservation + auth token:
  - `reservePort()` — binds TCP server to port 0 on 127.0.0.1, records OS-assigned port, closes

### Issue 8: Spawn sidecars as child processes — DONE
- Created `apps/desktop/src/sidecar.ts` — SidecarManager class for child process lifecycle:
  - `SidecarName` type: `'server' | 'terminal' | 'mcp'`
  - `SidecarManager` class tracks all spawned child processes with a Map
  - `spawn(name)` — spawns a child process with correct entry point and environment
  - `spawnServices()` — spawns terminal first (server depends on it), then server, with startup delays (Issue 9 replaces with health checking)
  - `killOne(name)` — graceful shutdown: SIGTERM first, SIGKILL after 2-second timeout
  - `killAll()` — kills all tracked sidecars
  - `killAllAndWait(timeoutMs)` — kills all and returns promise that resolves when all exit or timeout
  - `restart(name)` — kills and re-spawns a sidecar
  - `setExitHandler(handler)` — registers callback for unexpected exits (used by Issue 9)
  - `getLastStderr(name)` — returns ring buffer of last 50 stderr lines for crash diagnostics
  - `isRunning(name)` — checks if a sidecar process is still alive
  - `getProcess(name)` — returns the raw ChildProcess handle
- Entry point resolution:
  - Dev mode: resolves to TypeScript source (e.g., `packages/server/src/main.ts`) run via tsx
  - Production: resolves to bundled JS (e.g., `packages/server/dist/main.js`) run via `ELECTRON_RUN_AS_NODE=1`
- Environment construction:
  - Inherits process.env (with fixed PATH from fix-path.ts)
  - Overlays PORT, TERMINAL_PORT from reserved ephemeral ports
  - In production: sets DATA_DIR to app data directory, ELECTRON_RUN_AS_NODE=1
  - Filters out ELECTRON_RUN_AS_NODE and ELECTRON_RENDERER_PORT from inherited env
- stdout/stderr capture:
  - Line-by-line streaming via `readline.createInterface`
  - stdout logged at info level, stderr at warn level
  - stderr stored in a 50-line ring buffer per sidecar for crash diagnostics
- Updated `apps/desktop/src/main.ts`:
  - Imports and creates SidecarManager after port reservation
  - In production mode (!isDev): spawns services before creating window
  - In dev mode: services run via `turbo dev`, not spawned by Electron
  - Registers unexpected exit handler (logs for now, Issue 9 emits to renderer)
  - Graceful shutdown on `before-quit`, `SIGINT`, `SIGTERM`
  - Exports `getSidecarManager()` for use by other modules
