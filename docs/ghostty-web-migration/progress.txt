## Issue 6: Backend: Escape sequence buffering — COMPLETED

Date: 2026-03-14

### What was done
- Created `packages/terminal/src/lib/buffered-data-handler.ts` with `createBufferedDataHandler` function
- The function wraps raw PTY onData callbacks to hold back incomplete VT escape sequences:
  - Trailing bare ESC (`\x1b`) is held back
  - Trailing CSI introducer (`\x1b[`) is held back
  - Trailing incomplete CSI with parameters (`\x1b[38;5`) is held back via regex
  - Held-back fragments are prepended to the next chunk and forwarded when complete
  - Plain text and chunks ending with complete sequences pass through immediately
- Wired `createBufferedDataHandler` into `terminal-manager.ts` around both PTY data callbacks (spawn and restart)
- Created comprehensive test suite `packages/terminal/test/buffered-data-handler.test.ts` with 10 tests covering all acceptance criteria

## Issue 7: Backend: Headless terminal state manager — COMPLETED

Date: 2026-03-14

### What was done
- Added `@xterm/headless@^6.0.0` and `@xterm/addon-serialize@^0.14.0` as dependencies of `@laborer/terminal`
- Created `packages/terminal/src/lib/headless-terminal.ts` with `createHeadlessTerminalManager` factory:
  - Manages a `Map<string, HeadlessTerminalState>` of headless xterm instances
  - `create(terminalId, cols, rows, ptyWrite)` — creates a headless Terminal with SerializeAddon, wires `onData` to forward device query responses (DA1/DSR) back to the PTY
  - `write(terminalId, data)` — writes PTY output to the headless terminal to maintain parsed screen state
  - `getScreenState(terminalId)` — returns compact VT escape sequence serialization (~4KB) via SerializeAddon
  - `resize(terminalId, cols, rows)` — keeps headless terminal in sync with PTY dimensions
  - `dispose(terminalId)` / `disposeAll()` — cleans up xterm instances and event handlers
  - Re-creating with same ID auto-disposes the old instance (restart scenario)
- Integrated headless terminal manager into `terminal-manager.ts`:
  - Headless terminal created on spawn with matching cols/rows
  - All PTY output written to headless terminal alongside ring buffer and subscribers
  - Headless terminal resized in sync on resize RPC
  - Headless terminal re-created on restart
  - Headless terminal disposed on remove
  - All headless terminals disposed on shutdown
  - `getScreenState(terminalId)` exposed on the TerminalManager service interface for Issue 8 (WebSocket attach protocol) to use
- Created comprehensive test suite `packages/terminal/test/headless-terminal.test.ts` with 13 tests:
  - Creation and basic lifecycle (create, non-existent terminal)
  - Screen state serialization (written text, empty terminal)
  - Device query handling (DA1 response forwarded to PTY)
  - Resize (reflects in serialized state, no-op for non-existent)
  - Alternate screen mode (serialization includes `\x1b[?1049h`)
  - Dispose (clean disposal, no-op for non-existent, disposeAll)
  - Re-creation (old content cleared on restart)
  - Write no-op for non-existent terminal
