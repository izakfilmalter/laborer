### Issue 8: Spawn sidecars as child processes â€” DONE

  - `SidecarManager` class tracks all spawned child processes with a Map
  - `spawn(name)` â€” spawns a child process with correct entry point and environment
  - `spawnServices()` â€” spawns terminal first (server depends on it), then server, with startup delays (Issue 9 replaces with health checking)
  - `killOne(name)` â€” graceful shutdown: SIGTERM first, SIGKILL after 2-second timeout

  - `killAllAndWait(timeoutMs)` â€” kills all and returns promise that resolves when all exit or timeout
  - `restart(name)` â€” kills and re-spawns a sidecar
  - `setExitHandler(handler)` â€” registers callback for unexpected exits (used by Issue 9)
  - `getLastStderr(name)` â€” returns ring buffer of last 50 stderr lines for crash diagnostics
  - `isRunning(name)` â€” checks if a sidecar process is still alive
  - `getProcess(name)` â€” returns the raw ChildProcess handle
- Entry point resolution:
  - Dev mode: resolves to TypeScript source (e.g., `packages/server/src/main.ts`) run via tsx

- Environment construction:

  - Overlays PORT, TERMINAL_PORT from reserved ephemeral ports
  - In production: sets DATA_DIR to app data directory, ELECTRON_RUN_AS_NODE=1
  - Filters out ELECTRON_RUN_AS_NODE and ELECTRON_RENDERER_PORT from inherited env
- stdout/stderr capture:
