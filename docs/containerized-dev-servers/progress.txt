## Issue 4: LiveStore schema for container state — COMPLETED

Extended the `workspaces` LiveStore table to track container state and added `ContainerStarted`/`ContainerStopped` events with materializers.

### What was done:

**Table changes (schema.ts):**
- Added three nullable columns to `workspaces` table: `containerId`, `containerUrl`, `containerImage` (all `State.SQLite.text({ nullable: true })`)
- Workspace rows are created with null container fields by default
- Container fields are populated via `ContainerStarted` event and cleared via `ContainerStopped` event

**Event definitions (schema.ts):**
- `ContainerStarted` event: carries `workspaceId`, `containerId`, `containerUrl`, `containerImage` fields
- `ContainerStopped` event: carries `workspaceId` field
- Both events use the `Events.synced` pattern matching existing codebase conventions
- Events registered in the `events` object export

**Materializers (schema.ts):**
- `v1.ContainerStarted`: updates workspace row with `containerId`, `containerUrl`, `containerImage` via `workspaces.update({...}).where({ id: workspaceId })`
- `v1.ContainerStopped`: sets all three container fields to null via `workspaces.update({...}).where({ id: workspaceId })`
- `v1.WorkspaceCreated` materializer updated to always insert null for container fields (containers are started after workspace creation)

**Test updates:**
- Updated workspace lifecycle test assertion to include `containerId: null`, `containerUrl: null`, `containerImage: null`
- Added new test: "materializes container lifecycle events on the workspaces table" covering full container lifecycle (create workspace → start container → verify fields populated → stop container → verify fields cleared → verify other workspace fields preserved)
- Updated workspace management RPC test assertion to include container null fields

## Issue 5: ContainerService — create and destroy — COMPLETED

Created the `ContainerService` Effect service that manages Docker container lifecycle, and integrated it into the `WorkspaceProvider` create/destroy pipelines.

### What was done:

**New service (container-service.ts):**
- Created `ContainerService` as a `Context.Tag` class with tag `'@laborer/ContainerService'`
- Uses `Layer.effect` pattern, depends on `LaborerStore` for event commits
- `createContainer(params)` method:
  - Takes `workspaceId`, `worktreePath`, `branchName`, `projectName`, and `devServerConfig` (image, dockerfile, workdir)
  - Generates container name and URL via shared `containerName` utility
  - Runs `docker run -d --name {name} -v {worktreePath}:{workdir} -w {workdir} {image} sleep infinity`
  - Commits `ContainerStarted` event to LiveStore with containerId, containerUrl, containerImage
  - Returns `RpcError` with `CONTAINER_CONFIG_ERROR` if no image specified (Dockerfile builds not yet supported)
  - Returns `RpcError` with `CONTAINER_CREATE_FAILED` on docker run failure
- `destroyContainer(workspaceId)` method:
  - Looks up workspace in LiveStore to get container info
  - Derives container name from containerUrl
  - Runs `docker stop` then `docker rm` (best-effort: catches errors, logs warnings, continues)
  - Commits `ContainerStopped` event to LiveStore
  - Gracefully handles missing workspace or missing container (no-op with log)

**WorkspaceProvider integration (workspace-provider.ts):**
- Added `ContainerService` as a dependency (injected via `yield* ContainerService`)
- `createWorktree`: after workspace is committed to LiveStore, checks if `devServer.image` is configured. If so, calls `containerService.createContainer()`. On failure, performs full rollback (worktree removed, branch deleted, port freed, workspace removed from LiveStore).
- `destroyWorktree`: before worktree removal, checks if workspace has a `containerId`. If so, calls `containerService.destroyContainer()` with best-effort error handling (logs warnings, continues cleanup).

**Layer wiring:**
- `main.ts`: imported `ContainerService`, added `Layer.provide(ContainerService.layer)` in `HttpLiveBase` composition
- `test/rpc/test-layer.ts`: added `ContainerService.layer` to both `TestLaborerRpcLayer` and `TestLaborerRpcWithStoreLayer`
- `test/workspace-destroy-origin.test.ts`: added `ContainerService.layer` to `TestLayer`
