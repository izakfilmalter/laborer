## 2026-03-09: Issue 4 ŌĆö SidecarManager Rust module ŌĆö spawn and kill processes

Status: COMPLETED

### What was done
- Created `apps/web/src-tauri/src/sidecar.rs` ŌĆö a new Rust module containing:
  - **Shell environment probing**: Probes the user's login shell (`$SHELL` or `/bin/zsh`) using
    `env -0` (NUL-delimited output) to capture PATH and other variables that macOS GUI apps
    don't inherit. Tries interactive-login (`-il`) first, falls back to login-only (`-l`).
    Skips nushell (unsupported). 5-second timeout prevents hanging on slow shell configs.
  - **`SidecarManager` struct**: Stored in Tauri managed state. Probes shell env once at
    construction. Tracks spawned sidecars via `Mutex<Vec<TrackedSidecar>>`.
  - **`spawn()`**: Resolves sidecar binary path relative to `tauri::process::current_binary()`.
    Uses `tokio::process::Command` with `process-wrap` `ProcessGroup::leader()` on Unix for
    clean group-kill. Passes PORT, TERMINAL_PORT, DATA_DIR env vars merged with shell env.
    Pipes stdout/stderr to `log::info!`/`log::warn!` via async tokio tasks.
  - **Process lifecycle monitoring**: Async tokio task polls `child.try_wait()` and listens for
    kill signals via `mpsc::channel`. Emits `TerminatedPayload` (code + signal) via oneshot
    channel on exit.
  - **`kill_all()`**: Sends kill signal to all tracked sidecars and waits 500ms for cleanup.
  - **`take_exit_rx()`**: Allows taking the exit receiver for health check racing (used by Issue 5).
- Updated `apps/web/src-tauri/src/lib.rs`:
  - Added `mod sidecar;` declaration.
  - Creates `SidecarManager` (via `Arc`) before building the Tauri app.
  - Registers it as Tauri managed state via `.manage()`.
  - Changed `.run()` to `.build()` + `app.run()` to capture `RunEvent::Exit`.
  - On `RunEvent::Exit`, spawns a cleanup thread with a new tokio runtime to `kill_all()`.
- Updated `Cargo.toml` with new dependencies:
  - `tokio = { version = "1", features = ["process", "io-util", "time", "sync", "rt"] }`
  - `process-wrap = { version = "9", features = ["tokio1"] }`
- 7 unit tests covering: shell env parsing, env merging, nushell detection, SidecarName
  display/serialization.

  will be resolved when Issue 5/6 wire spawning into app setup)
- `cargo test` passes: 7/7 tests pass

## 2026-03-09: Issue 5 ŌĆö SidecarManager ŌĆö health checks and Tauri events

Status: COMPLETED

### What was done
- Extended `apps/web/src-tauri/src/sidecar.rs` with health check, state tracking, crash
  detection, Tauri event emission, and restart support:
  - **SidecarState enum**: `Starting`, `Healthy`, `Crashed { error }`, `Stopped` ŌĆö serializable
    as JSON with `#[serde(rename_all = "snake_case", tag = "status")]`.
  - **Health check polling** (`check_health`): Uses `reqwest` with a 2-second timeout, no-proxy
    for localhost. Polls at 100ms intervals after sidecar spawn.
  - **`spawn_and_wait_healthy()`**: New high-level method that spawns a sidecar and races three
    futures: health poll (100ms interval), process termination detection, and a 10-second timeout.
    Emits `sidecar:healthy` or `sidecar:error` Tauri events based on the outcome.
  - **Tauri event payloads**: `SidecarHealthyPayload { name }` for `sidecar:healthy`,
    `SidecarErrorPayload { name, error, last_stderr }` for `sidecar:error`.
  - **Stderr buffering**: Each sidecar maintains a ring buffer of the last 50 stderr lines
    (via `Arc<Mutex<Vec<String>>>`), included in error payloads for crash diagnostics.
  - **Crash detection** (`start_crash_monitor`): After a sidecar becomes healthy, a background
    tokio task monitors the exit receiver. If the process exits while in the `Healthy` state,
    it transitions to `Crashed` and emits `sidecar:error`.
  - **`kill_one()`**: Kills a specific sidecar by name, sets state to `Stopped`, waits 200ms.
  - **`restart()`**: Kills (if running) and re-spawns with full health check cycle.
  - **`restart_sidecar` Tauri command**: Exposed as `#[tauri::command]` for frontend invocation,
    takes a `SidecarName` parameter and delegates to `SidecarManager::restart()`.
  - **`get_state()`**: Public method to query current sidecar state.
- Updated `apps/web/src-tauri/src/lib.rs`:
  - Registered `restart_sidecar` in `invoke_handler`.
- Updated `Cargo.toml`:
  - Added `reqwest = { version = "0.12", default-features = false, features = ["rustls-tls"] }`.
- 15 unit tests covering: all existing tests + state serialization, payload serialization,
  health URL generation, health check failure for nonexistent server, state equality,
  manager empty state, get_state for unknown sidecar.

### Verification
- `cargo check` passes with no errors and no warnings
- `cargo test` passes: 15/15 tests pass
- No changes to TypeScript code
