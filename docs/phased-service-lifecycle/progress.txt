## Issue #14: Server deferred layer group (background initialization) ‚Äî COMPLETED

### What was done
- Created `packages/server/src/services/deferred-service.ts` with:
  - `SERVICE_INITIALIZING_CODE` ‚Äî sentinel error code (`'SERVICE_INITIALIZING'`) used in RpcError to signal that a deferred service is still initializing, detectable by the renderer
  - `serviceInitializingError(serviceName)` ‚Äî creates an RpcError with the SERVICE_INITIALIZING code
  - `makeServiceProxy<T>(serviceName, overrides?)` ‚Äî creates a Proxy that returns SERVICE_INITIALIZING errors for all method calls, with optional method overrides for services that need valid placeholder data (e.g., DockerDetection.check() returns `{ available: false }`)
  - `makeRefDelegatingService<Id, S>(tag, overrides?)` ‚Äî creates a `{ ref, proxy }` pair where:
    - `ref` is a `Ref<S>` holding the current implementation (starts with placeholder proxy)
    - `proxy` is a Proxy that delegates all method calls through the Ref to the current implementation
    - When the Ref is swapped to the real service, all future calls automatically go through to it
  - `DeferredServicesReady` ‚Äî Context.Tag holding a `Ref<boolean>` that tracks whether all deferred services have initialized (starts `false`, set to `true` when background fiber completes). Used by Issue #15 (Server "fully initialized" event)
  - `DeferredServicesReadyLayer` ‚Äî Layer.effect that provides DeferredServicesReady

- Restructured `packages/server/src/main.ts` for background initialization:
  - **DeferredServicesProxyLive** ‚Äî new `Layer.scopedContext` layer that:
    1. Creates 14 Ref-backed delegating proxies for all deferred service Tags that RPC handlers yield from
    2. DockerDetection gets a special placeholder override: `check()` returns `{ available: false }`
    3. Forks a single background fiber (`Effect.forkScoped`) that builds the full `DeferredServicesLive` layer graph
    4. Once built, swaps each Ref to the real service implementation from the built Context
    5. Sets `DeferredServicesReady` ref to `true` on success
    6. On failure: logs the error but does NOT crash the server ‚Äî RPCs continue returning SERVICE_INITIALIZING errors
    7. Returns a Context with all 14 proxy service implementations
  - **HttpLive** updated to use `DeferredServicesProxyLive` instead of `DeferredServicesLive` directly
  - The health endpoint responds as soon as core layers are built (before deferred services finish)

## Issue #16: Lazy sidecar connections (TerminalClient + FileWatcherClient) ‚Äî COMPLETED

### What was done
- **FileWatcherClient** (`packages/server/src/services/file-watcher-client.ts`) ‚Äî converted from eager to lazy connection:
  - Extracted `createFileWatcherRpcClient(url)` helper function for proper type inference of the RPC client
  - Layer construction (`Layer.scoped`) no longer calls `RpcClient.make()` or starts event stream subscription
  - Connection is established lazily on first method call (`subscribe`, `unsubscribe`, `updateIgnore`, `listSubscriptions`)
  - Uses a mutable closure variable (`cachedClient`) to cache the RPC client after first successful connection
  - Event stream subscription (`watcher.events()`) starts once after first successful connection, forked into the layer's scope
  - Retry with exponential backoff (1s base, up to 30s, max 5 retries) when sidecar is unavailable
  - All method errors mapped to `FileWatcherRpcError` for consistent error types
  - `onFileEvent()` handler registration works immediately (before connection) ‚Äî handlers stored in memory, dispatched when events arrive
  - Captured `layerScope` from `Effect.scope` during construction, provided to lazy `RpcClient.make()` via `Effect.provideService(Scope.Scope, layerScope)` so the client lifecycle is tied to the layer

- **TerminalClient** (`packages/server/src/services/terminal-client.ts`) ‚Äî converted from eager to lazy connection:
  - Extracted `createTerminalRpcClient(url)` helper function for proper type inference
  - Layer construction no longer calls `RpcClient.make()`, seeds the terminal map, or starts event stream subscription
  - Connection is established lazily on first method call via `getOrCreateClient` effect
  - Terminal map seeding (`terminal.list()`) happens after first successful connection
  - Event stream subscription (`terminal.events()`) starts once after connection, forked into layer's scope
  - Same retry pattern as FileWatcherClient (exponential backoff, max 5 retries)
  - `killAllForWorkspace()` gracefully returns 0 when sidecar is unreachable (catches connection error, logs warning)
  - `spawnContainerTerminal` and `spawnHostTerminal` get the RPC client lazily via `yield* getOrCreateClient`
  - `autoTypeScripts` and `scheduleAutoRun` now receive `rpcClient` as a parameter (instead of closing over it)
  - `buildAgentCommand` now takes `terminalPort` as parameter (instead of closing over `env`)
  - Environment variables (`TERMINAL_PORT`) cached in a closure variable after first lazy resolution

- **Test file** ‚Äî Created `packages/server/test/rpc/lazy-connections.test.ts` with 6 tests:
  1. FileWatcherClient layer builds instantly without connecting to file-watcher sidecar
  2. FileWatcherClient onFileEvent handler can be registered before connection
  3. TerminalClient layer builds instantly without connecting to terminal sidecar
  4. TerminalClient killAllForWorkspace returns 0 when no terminals tracked (before connection)
  5. FileWatcherClient layer builds in under 1 second without sidecar (timing test)
  6. TerminalClient layer builds in under 1 second without sidecar (timing test)

### Notes
- The lazy connection pattern uses `Effect.provideService(Scope.Scope, layerScope)` to tie the lazily-created RPC client's lifecycle to the layer's scope, ensuring proper cleanup on shutdown
- The event stream fibers are forked via `Effect.forkIn(layerScope)` (not `Effect.forkScoped`) since the fork happens outside the original `Layer.scoped` generator ‚Äî `forkIn` explicitly uses the captured scope
- Both clients maintain the same retry strategy as before (exponential backoff with spaced union) for the event stream subscription, but the initial connection retry is bounded (max 5 retries) to avoid blocking indefinitely
- The existing `DeferredServicesProxyLive` in `main.ts` still wraps these layers in Ref-backed proxies ‚Äî the lazy connection pattern means the background fiber that builds `DeferredServicesLive` completes faster since neither client blocks on sidecar connections during layer construction
- Issue #17 (Parallel sidecar spawning) is now unblocked
