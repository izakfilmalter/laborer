## Issue #2: Layout-utils: window tab operations ‚Äî COMPLETED

Date: 2026-03-13

### What was done
- Created new module `apps/web/src/panels/window-tab-utils.ts` with pure functions:
  - `addWindowTab(layout, tab?)` ‚Äî append a new empty window tab, makes it active
  - `removeWindowTab(layout, tabId)` ‚Äî remove a tab, switch active to nearest sibling (prefer right, fall back left)
  - `switchWindowTab(layout, tabId)` ‚Äî set active tab by ID
  - `switchWindowTabByIndex(layout, index)` ‚Äî set active tab by 1-based position (1-8, 9=last)
  - `switchWindowTabRelative(layout, delta)` ‚Äî cycle next/prev with wrapping
  - `reorderWindowTabs(layout, fromIndex, toIndex)` ‚Äî drag reorder
  - `findWorkspaceLocation(layout, workspaceId)` ‚Äî find which tab contains a workspace (searches nested tile trees)
  - `findTerminalLocation(layout, terminalId)` ‚Äî find tab + workspace + panel tab + pane for a terminal ID
  - `getWorkspaceTileLeaves(node)` ‚Äî collect all workspace tile leaves from a tile tree
  - `getAllWorkspaceTileLeaves(layout)` ‚Äî collect across all tabs
  - `getActiveWindowTab(layout)` ‚Äî get the active window tab
- Added exported types: `WorkspaceLocation`, `TerminalLocation`
- All functions are pure: input WindowLayout, output new WindowLayout

## Issue #3: Layout-utils: workspace tile operations ‚Äî COMPLETED

Date: 2026-03-13

### What was done
- Created new module `apps/web/src/panels/workspace-tile-utils.ts` with pure functions:
  - `addWorkspaceToTab(tab, workspaceId)` ‚Äî add a workspace as a new tile leaf; creates single leaf for empty tab, flattens into horizontal split for existing horizontal root, wraps in new horizontal split otherwise
  - `removeWorkspaceFromTab(tab, workspaceId)` ‚Äî remove a workspace tile by ID, collapses single-child splits, returns undefined layout when last workspace removed
  - `splitWorkspaceTile(tab, workspaceId, newWorkspaceId, direction)` ‚Äî split a workspace tile to add a new workspace beside it; same-direction splits are flattened (no unnecessary nesting), different-direction splits create nested split nodes
  - `resizeWorkspaceTiles(tab, nodeId, direction)` ‚Äî resize workspace tiles by walking up the tile tree to find a matching-orientation ancestor split, shifts 5% between the target and its sibling, enforces 5% minimum size
  - `reorderWorkspaceTiles(tab, workspaceOrder)` ‚Äî reorder workspace tiles based on workspace ID order; collects all leaves, reorders them, rebuilds as flat horizontal split; preserves panel tab state

### Tests
- 46 tests in `apps/web/test/workspace-tile-utils.test.ts` covering:
  - addWorkspaceToTab: empty tab, single workspace (creates horizontal split), existing horizontal split (flattens), vertical root (wraps), immutability
  - removeWorkspaceFromTab: empty layout, not found, only workspace (undefined), two-workspace collapse, three-workspace redistribute, nested collapse, deep cascade collapse, immutability
  - splitWorkspaceTile: empty tab, not found, root leaf horizontal/vertical, same-direction flattening (H‚ÜíH, V‚ÜíV), different-direction nesting (V‚ÜíH), nested workspace split, immutability
  - resizeWorkspaceTiles: empty tab, not found, root leaf, grow right/down, shrink left/up, minimum size enforcement, direction mismatch, edge (no sibling), nested resize, immutability
  - reorderWorkspaceTiles: empty tab, single workspace, two/three workspace reorder, unchanged order, partial order (appends unmatched), nonexistent IDs ignored, nested flatten, panel tab state preservation
  - Combined operations: add+remove round-trip, add+split+remove, add+reorder+resize
